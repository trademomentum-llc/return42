// Problem space

Connectivity is not a guarantee

Edge operations cannot depend on carrier networks or cloud backends. When WAN disappears, coordination must persist.

WAN dependency

Traditional IoT and telephony stacks assume continuous internet access. A single uplink failure silences the network.

Trust gaps

Local devices often operate with implicit trust. Elevated operations like SIP bridging need verified identity and capability.

Fragile gateways

A central bridge is a single point of failure. Without queue-and-replay, outages mean data loss.

// System architecture

Four planes, one resilient system

Each plane has a single responsibility. Trust gates separate local autonomy from elevated bridging.

◎ Local Mesh Plane

XIAO SOM nodes discover peers, exchange heartbeats, and route control messages over local RF/Wi-Fi without WAN.

MQTT/CoAP · mTLS · mesh routing

◉ Trust Plane

GSS-based identity bootstrap establishes trust levels and capability advertisements before high-risk operations.

GSS mock adapter · HMAC · capability tokens

◈ Gateway Plane

NUC gateway enforces policy, buffers queued events during outages, and reconciles state on recovery.

FastAPI · policy engine · Redis Streams

◇ SIP Plane

Conditional external session signaling through Asterisk or FreeSWITCH, enabled only for trusted devices and permitted carriers.

SIP/ARI · trust_level ≥ medium

// Security model

Trust is earned, not assumed

Every elevated operation is gated by a verified GSS trust context. Failure reduces the node to safe local-only mode.

  • Mutual identity verification via X.509 and GSS
  • Integrity and replay protection on all control commands
  • Capability advertisement based on trust level
  • Automatic local-only fallback on bootstrap failure
  • Append-only evidence logs for audit
